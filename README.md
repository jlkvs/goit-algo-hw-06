Завдання 2 
DFS-шлях: Алгоритм DFS обирає випадковий шлях у глибину до кінцевої вершини, не забезпечуючи найкоротшого шляху. Наприклад, DFS може знайти шлях із кількох кроків, який глибше просувається графом.
BFS-шлях: Алгоритм BFS, проходячи рівнями, гарантує, що перший знайдений шлях є найкоротшим (мінімальна кількість ребер між двома вузлами). Тому BFS-шлях, як правило, коротший за кількістю кроків.
Обґрунтування різниці
Глибина проти ширини: DFS рухається від вузла вглиб, доки не досягає кінцевої вершини або глухої точки, що іноді призводить до довших шляхів. У той час як BFS розглядає всі сусіди поточного вузла перед переходом на наступний рівень, що дозволяє швидше знайти оптимальний шлях.
Найкоротший шлях: BFS завжди знаходить шлях з мінімальною кількістю ребер між початковою і кінцевою точкою. У випадку соціальних мереж цей алгоритм є більш ефективним для знаходження "найкоротшого зв'язку" між людьми, оскільки він забезпечує мінімальну кількість посередників.
Висновок
Алгоритми DFS і BFS дають різні результати через різний підхід до обходу графа:
DFS підходить для задач, де важливо дослідити всі можливі варіанти зв’язків, не обов'язково мінімізуючи кроки.
BFS є оптимальним вибором для пошуку найкоротшого шляху між двома вузлами.
Для пошуку найкоротших шляхів у соціальних мережах доцільно використовувати алгоритм BFS, оскільки він швидко знаходить мінімальний шлях за кількістю ребер.
